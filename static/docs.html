Dotfiles Manager Documentation

Welcome to the comprehensive documentation for Dotfiles Manager - the complete solution for developer environment setup and management.

What is Dotfiles Manager?

Dotfiles Manager is a powerful command-line tool designed to streamline the process of setting up and managing development environments. It provides automation for:

  • Package installation via Homebrew
  • Dotfile management with GNU Stow
  • GitHub SSH key generation and setup
  • Developer onboarding workflows
  • Configuration sharing and discovery
Perfect for: New team members, bootcamp students, DevOps teams, data scientists, mobile developers, and anyone who wants to maintain consistent development environments across multiple machines.

Installation

Quick Install (Recommended)

brew tap wsoule/tap && brew install dotfiles

Alternative Installation

curl -fsSL https://raw.githubusercontent.com/wsoule/new-dotfiles/main/install.sh | bash

Manual Installation

You can also download the binary directly from the GitHub releases page:

# Download and install manually
wget https://github.com/wsoule/go-dotfiles/releases/latest/download/dotfiles-darwin-amd64
chmod +x dotfiles-darwin-amd64
sudo mv dotfiles-darwin-amd64 /usr/local/bin/dotfiles

Quick Start

New Developer Onboarding

The fastest way to get started is with the onboarding command:

dotfiles onboard --email your@email.com

This single command will:

  • Install essential development packages
  • Generate and configure SSH keys for GitHub
  • Set up basic development tools
  • Configure Git with your email

Using Templates

Start with a pre-built template for your development stack:

# Web development setup
dotfiles clone template:web-dev

# Data science environment
dotfiles clone template:data-science

# DevOps toolchain
dotfiles clone template:devops

Manual Setup

For custom configurations, start from scratch:

# Initialize new configuration
dotfiles init

# Add packages manually
dotfiles add git curl node python3

# Add applications
dotfiles add --cask visual-studio-code google-chrome

Developer Onboarding

The onboarding feature is designed to get new developers productive in minutes, not hours or days.

Basic Onboarding

dotfiles onboard --email developer@company.com

Onboarding with Custom Template

dotfiles onboard --email developer@company.com --template team-setup

What Happens During Onboarding

  1. Package Installation: Essential development tools are installed
  2. SSH Key Generation: New SSH key pair is created
  3. GitHub Integration: SSH key is added to GitHub account
  4. Git Configuration: User name and email are configured
  5. Dotfile Setup: Basic configuration files are installed
Team Setup: Organizations can create custom onboarding templates with company-specific tools and configurations to ensure consistency across the team.

Package Management

Dotfiles Manager provides comprehensive package management through Homebrew integration.

Adding Packages

# Add command-line tools
dotfiles add git curl wget tree jq

# Add GUI applications
dotfiles add --cask visual-studio-code google-chrome slack

# Add specific taps
dotfiles add --tap homebrew/cask-fonts

Removing Packages

# Remove packages
dotfiles remove wget tree

# Remove applications
dotfiles remove --cask slack

Listing Packages

# List all packages
dotfiles list

# List in JSON format
dotfiles list --json

# List specific type
dotfiles list --brews-only
dotfiles list --casks-only

Package Categories

Type Description Example
brews Command-line tools and libraries git, curl, python3, node
casks GUI applications visual-studio-code, chrome, slack
taps Third-party repositories homebrew/cask-fonts
stow Dotfile configurations vim, zsh, tmux

GitHub Integration

Seamless GitHub integration for SSH key management and repository access.

SSH Key Setup

# Generate and add SSH key to GitHub
dotfiles ssh --setup --email your@email.com

# Generate key only (manual GitHub setup)
dotfiles ssh --generate

Repository Cloning

# Clone configuration from GitHub user
dotfiles clone username

# Clone specific repository
dotfiles clone username/repo-name

# Clone from URL
dotfiles clone https://github.com/username/dotfiles

Configuration Sharing

# Share current configuration as GitHub Gist
dotfiles share gist --name "My Dev Setup" --description "Full-stack web development environment"

# Share with specific visibility
dotfiles share gist --private

Configuration Sharing

Share and discover development configurations with the community.

Sharing Your Configuration

# Share via CLI
dotfiles share gist --name="My Setup" --description="My awesome dev config" --author="yourname"

# Export to file first
dotfiles list --json > my-config.json

Finding Configurations

Browse community configurations on the main website or search by tags:

  • web-dev: Frontend and backend web development
  • data-science: Python, R, Jupyter environments
  • devops: Infrastructure and deployment tools
  • mobile-dev: iOS and Android development

Using Shared Configurations

# Clone from community
dotfiles clone https://dotfiles-web.com/api/configs/123

dotfiles clone config-url --merge

Basic Commands

Command Description Example
dotfiles init Initialize new configuration dotfiles init

Template Commands

Using Templates

# List available templates
dotfiles template list

# Use a template
dotfiles clone template:web-dev

# Use template with customization
dotfiles clone template:web-dev --add "docker,kubernetes"

Creating Templates

# Create template from current config
dotfiles template create --name "my-template" --description "My custom setup"

# Create template with inheritance
dotfiles template create --name "extended-web" --extends "web-dev" --add "docker"

Available Templates

  • web-dev: Full-stack web development (Node.js, Python, Git, VS Code)
  • data-science: Python, R, Jupyter, data analysis tools
  • devops: Docker, Kubernetes, Terraform, cloud tools
  • mobile-dev: iOS/Android development tools
  • minimal: Basic development essentials

Working with Templates

Templates provide pre-configured development environments for different use cases.

Template Structure

Templates are JSON files that define packages and configurations:

{
  "metadata": {
    "name": "Web Development Template",
    "description": "Full-stack web development environment",
    "author": "Your Name",
    "version": "1.0.0",
    "tags": ["web-dev", "frontend", "backend"]
  },
  "brews": ["git", "curl", "node", "python3", "yarn"],
  "casks": ["visual-studio-code", "google-chrome", "iterm2"],
  "taps": ["homebrew/cask-fonts"],
  "stow": ["vim", "zsh", "git"]
}

Template Inheritance

Templates can extend other templates:

{
  "extends": "web-dev",
  "addOnly": true,
  "brews": ["docker", "kubernetes-cli"],
  "casks": ["docker"]
}

Customization

Configuration File

Dotfiles Manager uses a JSON configuration file located at ~/.dotfiles/config.json:

{
  "brews": ["git", "curl", "tree"],
  "casks": ["visual-studio-code"],
  "taps": [],
  "stow": ["vim", "zsh"],
  "metadata": {
    "name": "My Development Setup",
    "author": "Your Name",
    "description": "Personal development environment",
    "created_at": "2024-01-01T00:00:00Z",
    "updated_at": "2024-01-01T00:00:00Z"
  }
}

Environment Variables

Variable Description Default
DOTFILES_DIR Configuration directory ~/.dotfiles
DOTFILES_REMOTE Default remote repository None
GITHUB_TOKEN GitHub API token None

Hooks and Scripts

Add custom scripts that run during installation:

# ~/.dotfiles/hooks/post-install.sh
#!/bin/bash

# Configure Git
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# Install additional tools
npm install -g typescript eslint

echo "Custom setup complete!"

Troubleshooting

Common Issues

Installation Fails

# Check Homebrew installation
brew --version

# Update Homebrew
brew update

# Fix permissions
sudo chown -R $(whoami) /usr/local/share/zsh

SSH Key Issues

# Check SSH key
ls -la ~/.ssh/

# Test GitHub connection
ssh -T git@github.com

# Regenerate SSH key
dotfiles ssh --generate --force

Configuration Not Found

# Check configuration location
ls -la ~/.dotfiles/

# Initialize new configuration
dotfiles init

# Restore from backup
dotfiles restore ~/backup.json

Debug Mode

# Run with debug output
dotfiles --debug install

# Verbose logging
dotfiles --verbose onboard --email test@example.com

API Reference

Dotfiles Manager provides a RESTful API for integration with other tools.

Base URL

https://dotfiles-web.com/api

Endpoints

Templates

Method Endpoint Description
GET /templates List all templates
GET /templates/{id} Get specific template
POST /templates Create new template
GET /templates/{id}/download Download template JSON

Configurations

Method Endpoint Description
GET /configs/search Search configurations
POST /configs/upload Upload configuration
GET /configs/stats Get usage statistics

Configuration Format

Schema

{
  "brews": ["string", "..."],
  "casks": ["string", "..."],
  "taps": ["string", "..."],
  "stow": ["string", "..."],
  "metadata": {
    "name": "string",
    "description": "string",
    "author": "string",
    "version": "string",
    "tags": ["string", "..."],
    "created_at": "ISO 8601 timestamp",
    "updated_at": "ISO 8601 timestamp"
  },
  "extends": "string (optional)",
  "overrides": ["string", "..."],
  "addOnly": "boolean",
  "public": "boolean",
  "featured": "boolean"
}

Field Descriptions

Field Type Description
brews Array Command-line packages from Homebrew
casks Array GUI applications from Homebrew Cask
taps Array Additional Homebrew repositories
stow Array Dotfile packages managed by GNU Stow
metadata Object Configuration metadata and information
extends String Parent template to inherit from
addOnly Boolean Only add packages, don't remove existing
